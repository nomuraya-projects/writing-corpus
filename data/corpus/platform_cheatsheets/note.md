# note.com マークダウンチートシート

**特徴**: 機能制限が多い。シンプルな記法のみ対応。

---

## 使用可能な記法

### 見出し

```markdown
# 見出し1
## 見出し2
### 見出し3
```

### リスト

```markdown
- 箇条書き
- 箇条書き

1. 番号付き
2. 番号付き
```

### 引用

```markdown
> 引用文
```

### リンク

```markdown
[テキスト](URL)
```

### 画像

エディタからアップロード（マークダウン記法は限定的）

### コードブロック

````markdown
```
コード
```
````

言語指定によるシンタックスハイライトは非対応

---

## 使用不可な記法

| 記法 | 状態 |
|------|------|
| テーブル | ❌ 使用不可 |
| 独自記法（:::message等） | ❌ なし |
| シンタックスハイライト | ❌ 言語指定不可 |
| 脚注 | ❌ 使用不可 |
| KaTeX数式 | △ 画像化される（非推奨） |

## 代替テクニック

### テーブルの代替

絵文字・記号でカテゴリ分け:

```markdown
📌 開発中 → ローカルサーバー
📌 単発処理 → File API
📌 配布アプリ → Electron/Tauri
```

または箇条書きで:

```markdown
- 良い点:
  - 項目1
  - 項目2
- 注意点:
  - 項目1
```

### 強調の代替

```markdown
❌ **強調テキスト**（使えるが読みづらい）

✅ 文章の流れで強調を表現
✅ 記号で目立たせる: ⚠️ 注意点
✅ 引用ブロックで囲む
```

### 注意喚起の代替

```markdown
⚠️ 注意: ここに注意事項

💡 ポイント: ここにポイント

📝 メモ: ここにメモ
```

### コードの代替（ハイライトなし対策）

コメントを多めに入れて可読性を確保:

```
// === ステップ1: 初期化 ===
const app = init();

// === ステップ2: 設定読み込み ===
const config = loadConfig();
```

---

## 執筆時の注意

1. **ビジュアル重視**: テキストだけでなく画像を活用
2. **絵文字で装飾**: 見出しや箇条書きに絵文字を添える
3. **シンプルに保つ**: 複雑な構造は避ける
4. **画像で補完**: 図表が必要な場合は画像として挿入
